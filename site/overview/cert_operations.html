Operating over certificates

Once a certificate has been issued by the method OrinocoX509Engine.generateCertificate the certificate has the status of valid. Certificates can be in the following status:

  • Valid: The certificated can be used. This is the status after generating it
  • Suspended: The certificate is not valid (appear in CRL or OCSP request as invalid) but it id possible to become valid again
  • Revoked: The certificate is invalid and there is not way to make it valid. It is a definitive state
  • Expired: The certificate is invalid, because the validity period is out of date

Get the status of the certificate

Get the status of a certificate is as easy as use the method OrinocoX509Engine.getCertificateStatus with the serial of the certificate.

@Autowired
OrinocoX509Engine engine;

...

// Get the status
CertificateStatus certificateStatus = engine.getCertificateStatus(certificateSerialNumber);

// Status is in certificateStatus property
certificateStatus.getCertificateStatus();

This method returns an CertificateStatus object. The most important method is getCertificateStatus which returns a CertificateStatusValues value. The possible values are:

  • V for valid
  • R for revoked
  • S for suspended
  • U for unknown
  • E for expired

Suspend a certificate

engine.suspendCertificate(certificateSerialNumber);

Restore a certificate

engine.restoreCertificate(certificateSerialNumber);

Revoke a certificate

engine.revokeCertificate(certificateSerialNumber);
